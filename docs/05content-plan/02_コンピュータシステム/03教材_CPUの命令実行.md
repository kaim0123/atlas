# 03 CPUの命令実行

命令が「どう1つずつ処理されるか」。性能（[04](04教材_CPUと性能.md)）の前に、仕組みを押さえる。

> 分割元: `00_分割元_CPUとメモリ.md` §2.1（拡張）／ 計画: [`ディレクトリ.md`](ディレクトリ.md) A-3

---

## 1. 命令サイクル

CPU は次の流れを繰り返します。この一連を**命令サイクル**といい、クロックに合わせて進みます。

```
フェッチ → デコード → 実行 → 書き戻し
（取り出し）（解読）  （演算） （結果格納）
```

| 段階 | 内容 |
|------|------|
| フェッチ | 主記憶から命令を取り出す |
| デコード | 命令を解読し、何をするか決める |
| 実行 | 演算装置で処理する |
| 書き戻し | 結果をレジスタや主記憶に格納する |

---

## 2. レジスタ

CPU 内部の高速な記憶。命令サイクルの各段階で使われます。

| レジスタ | 役割 |
|----------|------|
| 命令アドレスレジスタ（プログラムカウンタ） | 次に実行する命令のアドレスを保持 |
| 命令レジスタ | 取り出し中の命令そのものを保持 |
| 汎用レジスタ | 演算の対象や結果を一時的に保持 |

---

## 3. 割込み

実行中の処理を中断し、別の処理へ切り替える仕組み。OS の入口として概要のみ（詳しくは [11 プロセスとスケジューリング](11教材_プロセスとスケジューリング.md)）。

- **内部割込み**: 命令実行そのものが原因（0除算、ページフォールト等）。
- **外部割込み**: CPU の外が原因（入出力完了、タイマー、電源異常等）。

割込みが入ると、CPU は現在の状態を退避してから割込み処理へ移り、終わると元へ戻ります。

---

## 4. CPU の種類（概要）

| 用語 | 位置づけ |
|------|----------|
| RISC | 命令を単純・少数に絞り、高速化しやすい設計 |
| CISC | 複雑・多機能な命令を持つ設計 |
| GPU | 画像処理・並列演算に特化したプロセッサ |
