# コンピュータシステム ディレクトリ構成

**位置づけ**: e-Words「テクノロジ系 コンピュータシステム」（構成要素・ハードウェア・ソフトウェア・システム構成要素）を、初心者が段階的に学べる解説ページ群として再構成する計画。[`01_基礎理論/ディレクトリ.md`](../01_基礎理論/ディレクトリ.md) の直後、[`app構成.md`](../app構成.md) の `computer/` に対応する。

**ステータス**: 教材ドラフト1本（`01教材_CPUとメモリ.md`）・用語集4本（**計画専用・ページ化しない**）は執筆済み。`src/app/computer/` には半導体・メモリ・OS 等が先行実装済み。docs と app の対応付け・分割手順は下記「既存コンテンツの移行計画」に記載（**未着手**）。

**大方針**:
- **用語集4本は計画専用**。e-Words の棚卸し・漏れ確認に使うだけで、`src/app` に用語集ページは作らない。解説は教材ドラフト → `page.tsx` のみ。
- 用語集の全範囲を一度にカバーしない。**五大装置 → 半導体と回路 → CPU → メモリ → 入出力 → OS → システム構成**の順で、試験・実務の両方で頻出の土台を厚くする。
- 1ページは「1つの疑問に答える」粒度を目安にする。`01教材_CPUとメモリ.md` は CPU とメモリが同居しているため、実装時に分割する。
- 他セクションと重なる話題は**書き分け＋相互リンク**のみ（統合しない）。本計画は**このフォルダ内の教材ドラフトの整理**に留め、ネットワーク・インフラ・開発の詳細には踏み込まない。

**凡例**: `✓` 教材ドラフトあり / `＋` 新規執筆 / `……` はページ内トピック / `※app` は `src/app/computer/` に既存ページあり / 用語集は計画専用（app 化しない）

## 整理メモ

- **コンパクトなパス**: `computer/memory/virtual` のように2〜3階層まで。`computer/os/memory-management/paging-replacement` のような深いネストは避ける。
- **教材ファイル名**: 実装後も `docs/05content-plan/02_コンピュータシステム/` のドラフトをソースとして残し、`page.tsx` はそこから移植・分割する。app 先行のページは**逆輸出**で docs を追従させる。
- **試験対策と実務の両立**: 各ページ末尾に「この先どこで使うか」（例: 記憶階層 → `memory/speed`、仮想記憶 → `os/process`、クラスタ → `infra/virtualization`）を `RelatedLink` で1〜3本付ける。
- **上書き禁止**: `memory/`・`os/`・`semiconductor/` の既存 app 本文は、試験用語の追記以外では削らない。
- **用語集は計画専用**: e-Words との突合・漏れ確認に使う。サイト上の索引ページや用語リンクにはしない。
