## 全体マップ（学習順）

e-Words の4区分を、読者の理解の流れに並べ替えたもの。

```
コンピュータシステム（学習順）
│
├── A. 機械の仕組み          … 1台のコンピュータが「計算できる」理由と部品
│   ├── 全体像               五大装置・コンピュータの種類
│   ├── 半導体と論理         トランジスタ → 論理回路 → 加算器（CPUの下回り）
│   ├── CPU                  命令実行・性能・並列
│   ├── メモリ               記憶階層・主記憶・補助記憶
│   └── つなぎと入出力       バス・インタフェース・入出力装置
│
├── B. ソフトウェア          … ハードの上で動く OS とその管理機能
│   ├── OSの基礎             カーネル・モード・ジョブ管理の入口
│   ├── プロセス             プロセス・スレッド・スケジューリング
│   ├── 記憶管理             区画・スワッピング・仮想記憶・ページング
│   └── ファイルと保護       パス・権限・ファイルシステム
│
└── C. システム構成          … 複数機・処理形態・信頼性・評価
    ├── 処理形態と構成       集中・分散・CS・冗長・ストレージ構成
    ├── 信頼性               RASIS・フォールトトレラント・MTBF/MTTR
    └── 評価指標             レスポンス・スループット・ベンチマーク・TCO
```

**読了の目安**: A を終えれば「コンピュータの部品とデータの流れ」、B で「OS が何をしているか」、C で「システム全体の設計と評価用語」が説明できる。
