# 06 メモリと記憶階層

データの置き場所・速さ・容量のトレードオフ。OS の仮想記憶（[12](12教材_記憶管理と仮想記憶.md)）の前提。

> 分割元: `00_分割元_CPUとメモリ.md` §3 ／ 計画: [`ディレクトリ.md`](ディレクトリ.md) A-6

---

## 1. 記憶階層

CPU に近いほど**高速・小容量**、遠いほど**低速・大容量**。値段は高速なものほど高い。

```
CPU ← レジスタ ← キャッシュ ← 主記憶（メモリ）← 補助記憶（HDD/SSD）
      速い・小                              遅い・大
```

---

## 2. 主記憶とキャッシュ

| 用語 | 内容 |
|------|------|
| DRAM | 主記憶の主流。集積度が高く大容量化しやすいが、リフレッシュが必要。 |
| SRAM | キャッシュに使う。DRAM より高速だが容量は小さく高価。 |
| 揮発性 / 不揮発性 | 電源で消える（RAM）か、保持される（ROM・補助記憶）か。 |
| RAM / ROM | 読み書き可能な主記憶 / 読み出し中心の記憶。 |

**キャッシュの計算**（試験頻出）:

- **キャッシュヒット率**: アクセスしたデータがキャッシュにあった割合。
- **実効アクセス時間** ＝ ヒット率 × キャッシュのアクセス時間 ＋ (1 − ヒット率) × 主記憶のアクセス時間。
- **ライトスルー / ライトバック**: 書き込みを主記憶へ即時反映するか、後でまとめて反映するか。

---

## 3. 補助記憶

| 種類 | 特徴 |
|------|------|
| HDD | 磁気ディスク。大容量・単価が安いが、シークに時間がかかる。 |
| SSD | 半導体メモリ。ランダムアクセスが速い。 |
| 光学ディスク（CD/DVD/BD） | リムーバブルで長期保存向き。 |
| フラッシュメモリ（USBメモリ・SDカード） | 小型・不揮発。持ち運び用。 |

**ディスクの物理（概要）**: シリンダ・セクターで位置を指定。断片化を解消する操作がデフラグメンテーション。
