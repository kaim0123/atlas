# testing — テストの段階

> 分割元: `00_分割元_開発プロセス.md` §4。対応ページ: `sdlc/testing/levels/`

段階ごとに、範囲を広げながら検証します。

- **単体テスト**: 個々のモジュールや関数を検証。ドライバやスタブで周辺を模擬。
- **結合テスト**: モジュール間の連携を検証。トップダウン／ボトムアップなど。
- **システムテスト**: システム全体で機能・非機能要件を検証。性能・セキュリティテストを含む。
- **受入れテスト**: 発注者・利用者が「納品してよいか」を判断するための検証。
- **回帰テスト**: 変更後に、既存機能が壊れていないかを再テスト。

要件の取り違えを防ぐことと、テストの段階的な実施が重要です。

## ページ化メモ
- 単体（ドライバ・スタブ）→ 結合（トップダウン・ボトムアップ）→ システム・機能 → 受入れ・妥当性確認 → 回帰。
- テスト技法（網羅率・バグ曲線）は `sdlc/testing/techniques/` へ。
- 実務の Unit/Integration/E2E は `test/` へ RelatedLink。
