# overview — 開発の流れ（ライフサイクル）

> 分割元: `00_分割元_開発プロセス.md` §1。対応ページ: `sdlc/overview/`

システム開発は、要件を固めてから段階的に進みます。まず全体像を押さえます。

| 工程 | 主な活動 |
|------|----------|
| 要件定義 | 利害関係者から要求を聞き、**何を作るか**を文書化。機能要件・非機能要件を明確に。 |
| 設計 | 要件を実現する**仕組み**を決める。外部設計（画面・帳票）、内部設計（モジュール、DB物理設計）。 |
| 実装 | 設計に基づいてプログラムを作成。コーディング規約、単体テストをセットで。 |
| テスト | 単体→結合→システム→受入れと、範囲を広げながら検証。 |
| 導入・運用 | 本番リリース、教育、運用開始。 |
| 保守 | 是正保守（不具合修正）、適応保守（仕様変更）、予防保守。 |

**ウォーターフォール型**は「要件を固めてから設計・実装」。**アジャイル**では小さな単位で設計〜リリースを繰り返し、要求の変化に合わせて計画を見直します（詳細は `sdlc/process/`）。

## ページ化メモ
- 各工程の主な活動と成果物を概要表で示す。
- ウォーターフォールとアジャイルの違いは入口のみ。深掘りは process/ へ委譲。
