# requirements — 要件定義で押さえること

> 分割元: `00_分割元_開発プロセス.md` §2。対応ページ: `sdlc/requirements/`

「何を作るか」を決める段階。機能要件と非機能要件の区別が中心です。

- **機能要件**: システムが「何ができるか」（業務機能、画面、帳票）。
- **非機能要件**: 性能（応答時間、スループット）、可用性、セキュリティ、運用・保守のしやすさ。
- **ステークホルダ**: 発注者、利用部門、運用部門など。要件の合意には関係者の特定とコミュニケーションが不可欠。

## ページ化メモ
- 機能要件と非機能要件 / 性能要件（レスポンスタイム・スループット）。
- ステークホルダと合意 / システム要件定義とソフトウェア要件定義の違い（概要）。
- 要件の表現方法（ユースケース・DFD・ER図・UML入口）は `sdlc/requirements/modeling/` へ。
