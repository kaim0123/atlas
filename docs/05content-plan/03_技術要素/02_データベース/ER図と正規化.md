## 3. ER図と正規化

### 3.1 ER図

- **エンティティ**: 管理対象（顧客、商品、注文など）。
- **属性**: エンティティの性質。
- **リレーションシップ**: エンティティ間の関連（1対1、1対多、多対多）。

### 3.2 正規化

- **第1正規化**: 1セルに1つの値。繰り返し項目を別行に。
- **第2正規化**: 主キーの一部にだけ従属する属性を別表に（部分関数従属の解消）。
- **第3正規化**: 主キー以外の属性に従属する属性を別表に（推移関数従属の解消）。

正規化を進めると表が細かくなり結合が増えるため、検索性能とのバランスで非正規化することもあります。
