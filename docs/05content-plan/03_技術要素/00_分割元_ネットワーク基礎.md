# ネットワーク ① TCP/IPとIPアドレス

このノートでは、**階層モデル**、**TCP/IPの4層**、**IPアドレスとサブネット**、**NAT・DNS**を整理します。

---

## 1. 階層モデルの考え方

ネットワークは役割ごとに**階層**に分けて設計します。各層の役割が明確になり、変更の影響が局所化できます。

### 1.1 OSI基本参照モデル（7層）

| 層 | 名前 | 主な役割 |
|----|------|----------|
| 7 | アプリケーション層 | HTTP、メールなど |
| 6 | プレゼンテーション層 | データ形式・変換・暗号化 |
| 5 | セション層 | 通信の開始・終了・同期 |
| 4 | トランスポート層 | TCP/UDP、端末間の通信 |
| 3 | ネットワーク層 | 経路選択、IPアドレス |
| 2 | データリンク層 | 隣接機器間のフレーム、MAC |
| 1 | 物理層 | 電気・光の信号、ケーブル |

### 1.2 TCP/IPの4層

| 階層 | 主なプロトコル |
|------|----------------|
| アプリケーション層 | HTTP/HTTPS、SMTP、POP、DNS、DHCP、FTP |
| トランスポート層 | **TCP**（信頼性）、**UDP**（軽量） |
| インターネット層 | **IP**、ICMP、ARP |
| ネットワークインターフェース層 | イーサネット、無線LAN |

## 2. IPアドレスと経路

### 2.1 IPアドレス

- **IPv4**: 32ビット。例）192.168.1.1。アドレス不足のためIPv6へ移行中。
- **グローバルIP**: インターネット上で一意。
- **プライベートIP**: 組織内だけで一意。インターネットにはそのまま出せない。

### 2.2 サブネットマスクとCIDR

- **サブネットマスク**: ネットワーク部とホスト部の境目を示す。例）255.255.255.0。
- **CIDR表記**: 例）192.168.1.0/24 → 先頭24ビットがネットワーク部。

### 2.3 NATとDNS

- **NAT**: プライベートIPの端末がインターネットにアクセスするとき、送信元をグローバルIPに書き換える。少ないグローバルIPで多数の端末を接続できる。
- **DNS**: ドメイン名（www.example.com）からIPアドレスを調べる**名前解決**を行う。

---

## 3. TCPとUDP

- **TCP**: コネクション型。順序保証・再送で信頼できる転送。Web（HTTP）、メール送信に使う。
- **UDP**: コネクション不要。ヘッダが軽く、遅延を抑えたい用途（音声・動画、DNS問い合わせ）に使う。
- **ポート番号**: 同一IP上でどのアプリ向けか識別。80=HTTP、443=HTTPS、25=SMTPなど。
