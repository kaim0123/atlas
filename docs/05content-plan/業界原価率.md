# 業界別の原価率・キャッシュ・支払い ― コンテンツ計画

> **位置づけ**: 業界ごとに「数字のクセ」が違うことを、一覧表と仕組みで押さえる**横断リファレンス**ページ。  
> **想定パス**: `/finance/industry-cost`（会計・財務セクション内）  
> **nav 案**: 「決算書を読む（基礎）」の末尾、または「会社を分析する」の冒頭（`/finance/cost` の次、`/finance/analysis` の前）  
> **ステータス**: ＋ 新規執筆（app 未作成）

## このページで何を解決するか

同じ「会社」でも、飲食・小売・建設・IT では **原価率・手元資金の回り方・取引先との支払条件** がまるで違う。決算書の数字を読むとき、この差を知っているかどうかで解釈が変わる。

- [`/finance/cost`](../../src/app/finance/cost/page.tsx) … 固定費・変動費の**切り分け方**（カフェの例）
- [`/finance/metrics`](../../src/app/finance/metrics/page.tsx) … ROA・**CCC**（資金の回転日数）
- [`/finance/payroll`](../../src/app/finance/payroll/page.tsx) … 粗利益率と**損益分岐点**（業種別の数値例あり）
- [`/industry/*`](../../src/app/industry/page.tsx) … 各業界の**歴史と決算書の詳細**

本ページはその**橋渡し**。「業種を見たら、まずこの表を思い出す」ための地図。

**読者が持ち帰るもの**

1. 原価率・粗利益率・人件費率の**目安**（業界タイプ別）
2. なぜその業界は**キャッシュに苦労／余裕**があるのか（入金・支払・在庫・設備）
3. **CCC**（在庫＋売掛−買掛）と支払条件が業界構造にどう結びつくか
4. 各業界詳細ページへ飛ぶ**索引**

## §01 原価率だけ見ても足りない

### 用語の整理（表）

| 用語 | 意味 | 注意 |
|------|------|------|
| **原価率** | 売上原価 ÷ 売上 | 業界記事・現場でよく使う。**高い＝粗利が薄い** |
| **粗利益率** | 1 − 原価率（＝限界利益率に近い） | [`/finance/payroll`](../../src/app/finance/payroll/page.tsx) の損益分岐点計算に直結 |
| **FL比率**（外食） | （食材原価＋人件費）÷ 売上 | 原価率だけでは足りない。**L（人件費）**が別軸 |
| **人件費率** | 人件費 ÷ 売上 | サービス・医療・SI など**労働集約型**の生命線 |

### 要点

- 製造・小売は「原価率」、外食は「FL比率」、IT・コンサルは「人件費率＋粗利益率」が主役。
- PL の**どの段で急落するか**が業種のサイン（[`/finance/pl`](../../src/app/finance/pl/page.tsx) の階段比喩と接続）。
  - 売上総利益で落ちる → 原価・仕入が重い（小売・卸・建設）
  - 営業利益で落ちる → 固定費・人件費が重い（外食・SI・医療）

### Analogy 案

> **たとえるなら**: 原価率は「材料の重さ」、キャッシュ事情は「財布の中身の動き」、支払条件は「取引先との約束事」。体重（売上）が同じでも、背中に何を背負い、給料日と請求日がズレているかで、実際の生活（資金繰り）はまるで違う。

## §03 キャッシュの回り方 ― 3つの軸

### 軸1: 入金の速さ（売掛 vs 現金）

| パターン | 典型業界 | 意味 |
|----------|----------|------|
| **即日現金** | 外食・コンビニ | 売掛ほぼゼロ。日銭で仕入れを回せる |
| **月次売掛** | 製造・SI・卸 | 納品から入金まで 30〜90 日。資金が「宙に浮く」 |
| **前受金** | SaaS・不動分譲・学習塾 | **先にお金** → CF は PL より先行。契約負債（繰延収益） |

### 軸2: 在庫・設備（お金が「形」になる）

| パターン | 典型業界 | BS への表れ |
|----------|----------|-------------|
| **在庫** | 小売・製造・アパレル | 棚卸資産。死蔵在庫＝キャッシュの死 |
| **仕掛品・未成工事** | 建設・受託製造 | 進行基準。完成まで CF が出ていく |
| **設備** | 製造・鉄道・通信 | 有形固定資産。減価償却は CF に非キャッシュ |

### 軸3: CCC（キャッシュコンバージョンサイクル）

```
CCC ＝ 在庫回転日数 ＋ 売掛金回転日数 − 買掛金回転日数
```

| CCC の長さ | 典型 | 経営の意味 |
|------------|------|------------|
| **短い／マイナス** | 外食・コンビニ・Apple・Amazon・SaaS | 事業を回すほど手元資金が増えやすい |
| **長い** | 建設・受託 SI・在庫過多の小売 | 同じ利益でも**運転資金**が必要。成長＝資金不足リスク |

### Analogy 案

> **たとえるなら**: CCC は「お金の旅行日数」。出発（仕入・人件費の支払）から帰宅（売上の入金）まで何日かかるか。旅行が長い業界ほど、道中の宿泊費（運転資金）を用意しないと破綻する。[`/finance/cash`](../../src/app/finance/cash/page.tsx) の黒字倒産は、この旅行日数と返済のタイミングがズレたときに起きる。

### Diagram 案

横軸＝時間、縦軸＝キャッシュ残高の**3パターン波形**：

1. **外食型** … 毎日小さな入出金が細かく回る
2. **建設型** … 長期間マイナス → 完成時に大きくプラス（売掛回収）
3. **SaaS型** … 前受で先にプラス → サービス提供で PL 計上

## §04 支払いの考え方 ― 早く払う vs 遅らせる

### なぜ業界で支払条件が違うか

支払条件は「礼儀」ではなく**交渉力とリスク分担**の結果。

| 戦略 | 誰がやるか | 理由 |
|------|------------|------|
| **買掛を遅らせる** | 小売・大手メーカー | 在庫を売る前に代金を払わない。仕入先の運転資金を借りている |
| **前払い・現金** | 建設の下請・個人職人 | 取引先の倒産リスク。信用が薄いと現金要求 |
| **リース・割賦** | 外食・IT・航空 | 設備を**固定費化**。BS 上は使用権資産＋リース負債 |
| **前受・サブスク** | SaaS・塾・不動分譲 | 顧客の運転資金を借りる。CCC 改善 |

### 表: 支払条件と BS・CF への影響

| 条件 | BS | CF |
|------|----|----|
| 買掛 60 日 | 買掛金（負債）増 | 支払い前に CF プラス |
| 売掛 90 日 | 売掛金（資産）増 | 入金前に CF マイナス |
| 前受（SaaS 年契約） | 契約負債（繰延収益） | **先に CF プラス** |
| リース | 使用権資産＋リース負債 | 毎月のリース料（営業／財務 CF） |

### Aside 案

> 建設業の「下請けへの支払遅延」は、元請の CCC を短く見せるが、サプライチェーン全体の資金繰りを悪化させる。支払条件は**自分だけの問題ではない**。

## §05 診断に使う ― 表から決算書を読む

### ワークフロー（analysis への接続）

1. **PL** … 原価率・営業利益率で**業界タイプ**を推測（[`/finance/analysis`](../../src/app/finance/analysis/page.tsx) §02）
2. **BS** … 在庫・売掛・固定資産の比率で**商売の形**を特定（§03）
3. **CF** … 営業 CF が PL と乖離していないか。**黒字 CF 赤字**に注意
4. **CCC** … 業界平均と比べて**運転資金効率**を評価

### クイズ例（軽量・本文に 2 問）

**Q1**: 原価率 75%・営業利益率 3%・棚卸資産が大きい → どのタイプ？  
→ 小売または卸売。次に CCC と売上総資産回転率を見る。

**Q2**: 原価率 15%・契約負債が増加・営業 CF プラスだが営業利益マイナス → ？  
→ SaaS 成長期。[`/industry/software`](../../src/app/industry/software/page.tsx) の繰延収益パターン。

### CardGrid 案（3 枚）

1. **PL で業種を推測** … 原価率・利益率の水準感
2. **BS でモデルを特定** … 在庫・設備・売掛・前受
3. **CF で本当の体力** … 黒字倒産・成長赤字の見分け

## 執筆時の注意

- 数値は**丸めた目安**と明記。出典を載せるなら「一般的な経営書・業界平均のレンジ」程度に留める（個別企業の決算で補足）。
- 「キャッシュ」は口語の**手元資金・資金繰り**と、会計の**CF**の両方に触れるが、混同しない（Hero か §03 冒頭で一言定義）。
- 金融・医療は原価率フレームが効かない → **別枠**として表で明示し、無理に比較しない。
- IT は **ソフトウェア vs SI** を分ける（粗利率もキャッシュも真逆に近い）。
- 実装時は [`src/app/finance/payroll/page.tsx`](../../src/app/finance/payroll/page.tsx) の業種別原価率表と**数値を揃える**（飲食 30%・卸 90% 等）。
