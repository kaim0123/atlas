# 基礎理論 ディレクトリ構成

**位置づけ**: e-Words「テクノロジ系 基礎理論」を、初心者が段階的に学べる解説ページ群として再構成する計画。[`app構成.md`](../app構成.md) のフェーズ1の**最上流**に置く（コンピュータ・ネットワークより先に読む前提）。

**ステータス**: 教材ドラフト4本・用語集2本は執筆済み。`src/app` への実装・索引ページ・`nav.ts` 追記は未着手。

**大方針**:
- 用語集の全範囲を一度にカバーしない。**離散数学 → 確率・統計 → 情報理論 → アルゴリズム**の順で、試験・実務の両方で頻出の土台を厚くする。
- 1ページは「1つの疑問に答える」粒度を目安にする。教材ドラフトが長い箇所（③確率統計と情報理論など）は分割を検討する。
- 他セクションと重なる話題は**書き分け＋相互リンク**のみ（統合しない）。

**凡例**: `✓` 教材ドラフトあり / `＋` 新規執筆 / `……` はページ内トピック

```
Atlas

※ フェーズ1（computer/ 以降）の各ページから、必要な theory/ ページへリンクで戻る。
```

**想定 URL プレフィックス**: `/theory/...`（`src/app/theory/`）

## 1. numbers/ — 数の表現

コンピュータが数をどう記録・演算するか。以降のすべて（メモリ、論理回路、浮動小数点バグ）の前提。

```
theory/numbers/
├── page.tsx              ✓ ← 01教材_数と基数変換.md
│   …… 位取り記数法 / 2・8・16進 / 基数変換 / 2の補数 / 固定小数点の概要
└── floating/             ＋
    └── page.tsx          浮動小数点と演算精度
        …… IEEE754の考え方 / 仮数・指数 / 丸め誤差 / オーバーフロー・アンダーフロー / 桁落ち
```

**教材①との境界**: 補数・固定小数点は `numbers/page.tsx` に残す。浮動小数点の詳細だけ `floating/` へ切り出す。

---

## 2. logic/ — 論理

if 文・条件式・デジタル回路の共通言語。

```
theory/logic/
└── page.tsx              ✓ ← 02教材_論理と真理値表.md
    …… 命題と論理式 / AND・OR・NOT・XOR / 真理値表 / ド・モルガンの法則 / 論理回路との対応
```

**将来の拡張（＋）**: 集合・ベン図・命題論理の応用は、初学者向けには `logic/page.tsx` 内の1見出しで十分。独立ページ化は需要を見てから。

---

## 3. probability/ — 確率・統計

信頼性計算・データの読み方・仮説検定の入口。

```
theory/probability/
├── page.tsx              ✓ ← 03教材_確率統計と情報理論.md（§1〜§2 確率・統計部分）
│   …… 事象と確率 / 加法定理・乗法定理 / 条件付き確率 / 代表値 / 分散・標準偏差
└── inference/            ＋
    └── page.tsx          推定と仮説検定
        …… 統計的推定 / 回帰の概要 / 仮説検定 / 有意水準・p値（試験頻出のみ）
```

**分割理由**: 教材③は確率・統計と情報理論が同居している。初学者は「確率の計算」と「データの読み方」を先に、推定・検定は後半に回すと負荷が下がる。

---

## 4. information/ — 情報理論

データの符号化・圧縮・文字の表現。`dev/`・`network/` への橋渡し。

```
theory/information/
├── page.tsx              ✓ ← 03教材_確率統計と情報理論.md（§3 情報理論部分）
│   …… 情報量 / エントロピー / ハフマン符号 / データ圧縮の考え方
└── encoding/             ＋
    └── page.tsx          文字コード
        …… ASCII / Unicode / 文字化けの原因 / バイト列と表示の対応
```

**他セクションとの境界**:
- `network/` の誤り検出・CRC・パリティ → 通信の実務側。`information/` では「なぜ冗長ビットが要るか」の理論のみ触れ、詳細はリンク。
- `dev/language-basics/` の構文解析 → 字句・構文の実装側。`formal/` では文法の**記述法**に留める。

---

## 5. algorithms/ — アルゴリズム

データの置き方と、整列・探索・計算量。`dev/`・`design/` の前提。

```
theory/algorithms/
├── structures/           ✓ ← 04教材_アルゴリズムとデータ構造.md（§1〜§2）
│   └── page.tsx          データ構造
│       …… 配列 / リスト / スタック / キュー / 木（2分木・2分探索木）
├── sorting/              ✓ ← 04教材（§3 ソート）
│   └── page.tsx          ソートアルゴリズム
│       …… バブル・選択・挿入 / マージ・クイック・ヒープ / 安定ソート
├── search/               ✓ ← 04教材（§4 探索）
│   └── page.tsx          探索アルゴリズム
│       …… 線形探索 / 2分探索 / ハッシュ探索の概要
├── complexity/           ＋
│   └── page.tsx          計算量とオーダー記法
│       …… O記法 / 最悪・平均・償却 / アルゴリズム選択の判断基準
└── graphs/               ＋
    └── page.tsx          グラフと最短経路
        …… 有向・無向グラフ / 深さ優先・幅優先 / ダイクストラ法の概要 / 動的計画法への入口
```

**分割理由**: 教材④は1ファイルにまとまっているが、初学者は「構造」「整列」「探索」「計算量」を別々に復習したい。`structures/` → `sorting/` → `search/` → `complexity/` の順が自然。

**他セクションとの境界**:
- 用語集「アルゴリズムとプログラミング」の**プログラミング・言語**（変数、制御構造、OOP、HTML 等）→ `dev/language-basics/` 側。`theory/algorithms/` ではアルゴリズムとデータ構造のみ。

## 6. formal/ — 形式言語（発展）

コンパイラ・正規表現・構文の理論。必修ではないが、言語処理を深く学ぶ人向け。

```
theory/formal/
└── page.tsx              ＋
    …… 形式言語 / BNF・構文図式 / 正規表現 / 有限オートマトン / 逆ポーランド表記
```

**優先度**: フェーズ0の最後。`dev/tooling/`・`dev/language-basics/` を読む前の「予習」として置くか、並行リンクにする。

## 7. 用語集（索引からリンク）

解説ページ本体とは別枠。e-Words 一覧＋自分用メモ。各トピックページから `[[用語]]` 相当で相互参照。

| ファイル | 扱う範囲 | 対応する解説ページ |
|---|---|---|
| `用語集_基礎理論.md` | 離散数学・応用数学・情報理論・通信・計測制御 | §1〜§4 各 `theory/*` |
| `用語集_アルゴリズムとプログラミング.md` | データ構造・アルゴリズム（§1〜§2） | `theory/algorithms/*` |

**用語集のみで、当面独立ページを作らない領域**（他セクションへ委譲）:

| 領域 | 委譲先 |
|---|---|
| 通信理論（変復調・多重化・誤り訂正の実装） | `network/` |
| 数値計算・線形代数・数値解析 | 必要になったら `theory/numerical/` を新設（優先度低） |
| 待ち行列理論 | 必要になったら `theory/queueing/`（優先度低） |
| AI・機械学習 | 必要になったら `theory/ml/` または独立セクション |
| コンパイラの実装（字句解析〜コード生成） | `dev/language-basics/` |
| プログラミング言語・マークアップ | `dev/` |

## 整理メモ

- **コンパクトなパス**: `theory/numbers` のように2階層まで。`theory/discrete-mathematics/radix-conversion` のような深いネストは避ける。
- **教材ファイル名**: 実装後も `docs/05content-plan/01_基礎理論/` のドラフトをソースとして残し、`page.tsx` はそこから移植・分割する。
- **試験対策と実務の両立**: 各ページ末尾に「この先どこで使うか」（例: 補数 → `computer/memory`、情報量 → `network/`）を `RelatedLink` で1〜3本付ける。
